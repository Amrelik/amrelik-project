Урок 9. Поиск путей в графах

Во многих практических задачах (прокладка маршрутов, проектирование сетей, оптимизация логистики) требуется находить пути между вершинами графа, оптимальные по некоторому критерию. В этом уроке мы рассмотрим основные алгоритмы поиска кратчайших путей и построения минимальных остовных деревьев.

1. Основные понятия

Путь в графе — последовательность вершин, в которой каждая следующая соединена с предыдущей ребром. Длина пути может измеряться числом рёбер (для невзвешенных графов) или суммой весов рёбер (для взвешенных). Кратчайший путь — путь минимальной длины между заданными вершинами.

В зависимости от свойств графа (наличие весов, отрицательные рёбра, циклы) применяются разные алгоритмы.

2. Поиск в ширину (BFS) для невзвешенных графов

Если все рёбра имеют одинаковый вес (или вес не учитывается), кратчайший путь по числу рёбер находится с помощью алгоритма поиска в ширину (Breadth‑First Search). Алгоритм обходит граф слоями: сначала стартовую вершину, затем всех её соседей, потом соседей соседей и т.д. Первое обнаружение целевой вершины гарантирует кратчайший путь.

Пример. Найти кратчайший путь от A до F в невзвешенном графе: A соединена с B и C; B с D; C с D и E; D с F; E с F.

Решение: BFS из A: слой 0: A; слой 1: B, C; слой 2: D (из B и C), E (из C); слой 3: F (из D и E). Кратчайший путь A→C→E→F (длина 3).

3. Алгоритм Дейкстры (положительные веса)

Для взвешенных графов, где веса рёбер неотрицательны, классическим методом является алгоритм Дейкстры. Он находит кратчайшие расстояния от одной вершины до всех остальных.

Идея алгоритма

  1. Каждой вершине присваивается предварительное расстояние: 0 для стартовой, ∞ для остальных.
  2. Все вершины считаются необработанными. Пока есть необработанные вершины, выбирается вершина с минимальным известным расстоянием (текущая).
  3. Для каждого соседа текущей вершины вычисляется новое расстояние как сумма расстояния до текущей и веса ребра. Если это расстояние меньше текущего известного расстояния до соседа, оно обновляется.
  4. Текущая вершина помечается как обработанная (больше не рассматривается).

Пример. Дан взвешенный граф: вершины 1,2,3,4. Рёбра: 1-2 (вес 4), 1-3 (2), 2-3 (1), 2-4 (5), 3-4 (8). Найти кратчайшие расстояния от вершины 1 до всех остальных.

Решение: Стартуем с 1 (расст. 0). Обрабатываем 1: соседи 2 (0+4=4) и 3 (0+2=2). Запоминаем d[2]=4, d[3]=2. Минимальная среди необработанных — 3 (2). Обрабатываем 3: сосед 2 (2+1=3) — это меньше 4, обновляем d[2]=3; сосед 4 (2+8=10). Теперь минимальная необработанная — 2 (3). Обрабатываем 2: сосед 4 (3+5=8) — меньше 10, обновляем d[4]=8. Обрабатываем 4 — изменений нет. Итог: d[1]=0, d[2]=3, d[3]=2, d[4]=8.

Алгоритм Дейкстры работает только при неотрицательных весах. Если есть отрицательные рёбра, он может дать неверный результат.

4. Алгоритм Беллмана–Форда (отрицательные веса)

Этот алгоритм находит кратчайшие пути от одной вершины до всех остальных даже при наличии отрицательных весов, но не терпит отрицательных циклов (циклов с отрицательной суммой весов). Он также может обнаруживать такие циклы.

Алгоритм выполняет N-1 итерацию (N — число вершин), на каждой итерации пытается улучшить расстояния, используя все рёбра. После N-1 итерации дополнительная проверка позволяет выявить отрицательный цикл.

Пример (упрощённо). Для графа с вершинами A, B, C и рёбрами: A→B (4), A→C (2), B→C (–3). Запуск из A: после первой итерации d[B]=4, d[C]=2. Вторая итерация: через B можно улучшить C: 4 + (–3)=1 < 2, обновляем d[C]=1. Третья итерация изменений не даст. Результат: A→C кратчайший путь со стоимостью 1.

5. Поиск всех путей или максимального пути

Иногда требуется найти не кратчайший, а, например, все возможные пути или путь с максимальной пропускной способностью. Для этого применяются модификации обходов или специальные алгоритмы (Флойда–Уоршелла, алгоритм Йена для k кратчайших путей). В школьном курсе обычно ограничиваются кратчайшими путями и минимальным остовом.

6. Минимальное остовное дерево (МОД)

Остовное дерево — это подграф, содержащий все вершины исходного связного графа и являющийся деревом (без циклов). Минимальное остовное дерево — остовное дерево с наименьшей возможной суммой весов рёбер. Задачи такого типа возникают при прокладке коммуникаций (дорог, кабелей) с минимальными затратами.

6.1. Алгоритм Прима

Строит МОД, начиная с произвольной вершины, и постепенно добавляет самое дешёвое ребро, соединяющее построенное дерево с ещё не включённой вершиной.

Пример. Граф с вершинами 1–4 и весами: (1,2)=5, (1,3)=3, (2,3)=2, (2,4)=4, (3,4)=6. Начинаем с 1. Добавляем ребро (1,3) вес 3. Множество вершин {1,3}. Минимальное ребро, соединяющее это множество с остальными: (1,2)=5 и (3,2)=2 — выбираем (3,2) вес 2, добавляем вершину 2. Теперь вершины {1,2,3}. Доступные рёбра: (2,4)=4 и (3,4)=6 — берём (2,4) вес 4. Получили остов с суммой 3+2+4=9.

6.2. Алгоритм Краскала

Сортирует все рёбра по возрастанию веса и последовательно добавляет рёбра, не образующие цикла, пока не будут соединены все вершины. Для проверки циклов используется структура данных «система непересекающихся множеств» (DSU).

Пример на том же графе. Сортируем рёбра: (2,3)=2, (1,3)=3, (2,4)=4, (1,2)=5, (3,4)=6. Добавляем (2,3) — нет цикла. Добавляем (1,3) — нет цикла. Добавляем (2,4) — нет цикла. Все вершины соединены. Сумма 2+3+4=9 (та же).

7. Задания для самостоятельной работы

Задание 1. На невзвешенном графе (вершины A,B,C,D,E) заданы рёбра: AB, AC, BD, CD, DE. Найдите кратчайший путь из A в E, используя BFS.

Задание 2. Дан взвешенный граф с вершинами 1,2,3,4,5. Веса рёбер: (1,2)=7, (1,3)=9, (1,4)=14, (2,3)=10, (2,5)=15, (3,4)=2, (3,5)=11, (4,5)=6. Найдите кратчайшие расстояния от вершины 1 до всех остальных алгоритмом Дейкстры.

Задание 3. В графе из задания 2 постройте минимальное остовное дерево алгоритмом Краскала. Чему равна сумма весов?

Задание 4 (с отрицательными весами). Граф: вершины A,B,C. Рёбра: A→B (4), B→C (–2), C→A (–1). Найдите кратчайшее расстояние от A до C алгоритмом Беллмана–Форда. Есть ли отрицательный цикл?

Задание 5 (практическое). Используя матрицу весов из практической работы №15 (или придумайте граф из 5 вершин с произвольными весами), вычислите длину минимального остова по алгоритму Прима.

Ответы для самопроверки

Задание 1. A→C→D→E (длина 3).

Задание 2. d[1]=0, d[2]=7, d[3]=9, d[4]=11 (через 3: 9+2=11), d[5]=17 (через 4: 11+6=17).

Задание 3. Сортируем рёбра: (3,4)=2, (4,5)=6, (1,2)=7, (3,5)=11 (можно не брать), (2,3)=10 и т.д. После добавления (3,4), (4,5), (1,2) имеем вершины {3,4,5} и {1,2} — несвязно. Добавляем (2,3)=10, соединяем. Сумма = 2+6+7+10 = 25. Можно проверить другие варианты.

Задание 4. После двух итераций: d[A]=0, d[B]=4, d[C]=2. Третья итерация: через C→A даёт 2+(-1)=1 < 0? Нет, для A это хуже. Проверка на отрицательный цикл: проход по всем рёбрам ещё раз — A→B (0+4=4) не улучшает B (4), B→C (4-2=2) не улучшает C (2), C→A (2-1=1) улучшает A (было 0? нет, 0 меньше 1, значит нет улучшения). Отрицательного цикла нет. Кратчайший путь A→B→C = 4-2=2.

Задание 5. Ответ зависит от выбранной матрицы.
